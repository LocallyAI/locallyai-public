# LocallyAI Manager — desktop wrapper

A minimal macOS WKWebView wrapper around the firm's Manager UI. Replaces
"open the URL in Safari" with a real app icon a non-technical user can
double-click. Built so lawyers and DPOs don't need to type or remember a
URL.

## Why this exists

The Manager UI lives at `https://<firm-office-mac>:8000` and serves the
React app from the FastAPI backend. That works for engineers. It does
not work for lawyers, who are not going to type "https://" anything.
This app gives them a regular macOS dock icon that opens straight into
the firm's deployment.

The implementation is intentionally small (~200 lines of Swift):

- A `WKWebView` pointed at the deployment URL
- Accepts the self-signed cert at the navigation-delegate level
  (every firm runs a self-signed cert generated by `install.sh`)
- Persistent cookie / localStorage so the admin key survives restarts
- "Set Manager URL…" menu item for changing the target deployment
- Friendly error page if the office Mac isn't reachable

## Building

Requires Xcode Command Line Tools (`xcode-select --install`) for `swiftc`.

```bash
# Default URL: https://office-mac.local:8000
./build.sh

# Per-firm build with the firm's hostname baked in
MANAGER_URL=https://office.acme-law.local:8000 ./build.sh
```

Output:

- `dist/LocallyAI Manager.app` — the bundle to drag-and-drop to `/Applications`
- `dist/LocallyAI Manager.zip` — for distribution (AirDrop / email / `/admin/installers/`)

## Distribution

Two paths:

### Manual (dogfood / one-off)

1. Build with the firm's URL baked in (see above)
2. AirDrop or email the `.zip` to each staff laptop
3. User unzips, drags to `/Applications`
4. **First launch only**: right-click → Open (Gatekeeper warns on ad-hoc-signed
   apps; clicking "Open" bypasses it once)
5. Lawyer pastes admin key into LoginGate
6. Done

### Via `/admin/installers/` endpoint (production)

The api.py exposes `GET /admin/installers/` which serves pre-built client
installer artifacts. To make this app available there:

1. Run `./build.sh` with the firm's URL
2. Move `dist/LocallyAI Manager.zip` into `~/locallyai/storage/installers/`
3. Restart the API (or POST `/admin/installers/refresh`)
4. The Manager UI's "Client Apps" panel will surface it for the firm's IT
   to redistribute

In a real release pipeline this gets a proper Developer ID signature +
notarisation; the ad-hoc sign here is fine for dogfood + manual install
but Apple won't let it open without right-click on first use.

## Configuring the URL after install

The URL precedence is:

1. `LOCALLYAI_MANAGER_URL` environment variable (debug override)
2. `UserDefaults` key `ManagerURL` (set via "Set Manager URL…" menu)
3. Compile-time default (baked at build time via `MANAGER_URL=...`)

So a user with a working install can change the target deployment
through the menu — useful if the firm's office Mac hostname changes
(failover, move to new office, etc.).
