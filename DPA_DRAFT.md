## 1. Definitions

1.1 "Personal Data", "Data Subject", "Processing", "Controller", "Processor" carry the meanings given in the UK GDPR and the Data Protection Act 2018.

1.2 "Services" means the private AI inference and document retrieval platform supplied by the Processor under the Order Form dated [DATE].

1.3 "Client Data" means all data, documents, queries, and outputs submitted to or generated by the Services by the Controller or its authorised users.

1.4 "Hardware" means the dedicated computing equipment installed by the Processor at the Controller's premises as described in Schedule 1.

---

## 2. Scope and purpose

2.1 The Processor will process Client Data solely for the purpose of providing the Services as described in the Order Form and in accordance with the Controller's documented instructions.

2.2 The Processor will not process Client Data for any other purpose, including training, fine-tuning, benchmarking, or improving any AI model.

2.3 If the Processor is required by applicable law to process Client Data for any other purpose, it will inform the Controller before doing so (unless prohibited by law).

---

## 3. Data location and residency

3.1 **All Client Data is processed exclusively on the Hardware installed at the Controller's premises.** No Client Data is transmitted to, stored on, or processed by any system outside the Controller's physical environment.

3.2 The Processor has no remote access to Client Data. The Processor may access system health telemetry (CPU load, RAM usage, disk capacity, uptime) solely for the purpose of infrastructure monitoring. This telemetry contains no Client Data, no query content, and no document content.

3.3 The Controller retains physical and logical control of the Hardware at all times.

---

## 4. Shared liability and compliance undertakings

4.1 The Processor acknowledges that the Controller is a regulated entity subject to the Solicitors Regulation Authority (SRA) Code of Conduct, the UK GDPR, and applicable legal professional privilege rules.

4.2 The Processor undertakes that the architecture and operation of the Services is designed to preserve legal professional privilege and to comply with the Controller's obligations under the above regulations.

4.3 In the event of a data incident arising from a defect in the Processor's software, configuration, or Hardware (as installed by the Processor), the Processor accepts shared liability for the consequences of that incident up to the limit specified in Clause 9.

4.4 The Processor will cooperate fully with any regulatory investigation or audit relating to the Services, including providing audit trail documentation on request within 5 business days.

---

## 5. Audit trail

5.1 The Services maintain a complete, tamper-evident audit log of every query made to the system, including: timestamp, user identifier (hashed), query hash (SHA-256), model used, documents referenced, and retrieval time.

5.2 The audit log contains no raw query content. It records the existence and metadata of each query, not its substance.

5.3 The Controller may request an audit trail export in PDF or CSV format at any time via the administration interface or by written request to the Processor. The Processor will provide the export within 2 business days.

5.4 Audit logs are retained for a minimum of 6 years from the date of creation.

---

## 6. Sub-processors

6.1 The Processor will not engage any sub-processor to process Client Data without prior written consent from the Controller.

6.2 Given the on-premises architecture, **no sub-processor has access to Client Data** (defined as documents, queries, responses, audit content, user identifiers, or any other Personal Data processed on the Hardware). The list of upstream services the Processor uses, what data each can observe, and the affirmative carve-out for each, is:

| Upstream | Role | What they can observe | Client Data exposure |
|---|---|---|---|
| Cloudflare | Worker hosting (vendor monitor + kill switch) | Anonymised heartbeats containing `firm_id` (SHA-256 hash of firm name, one-way), `node_id` (deployment hostname), health gauges (boolean / numeric), structured alert codes, macOS/Python/backend version strings | **None.** Heartbeats explicitly exclude document content, user names, queries, responses, audit entries, billing entries. The `firm_id` is a hash; the firm name itself is not transmitted. |
| GitHub | Code repository hosting + signed release tag distribution | Source code (which contains no Client Data) and commit metadata (vendor identities, not firm) | **None.** |
| Hugging Face | Anonymous public model + embedder downloads at install time | Source IP of the download (a single transaction per install) | **None.** No account, no firm-attributable identifier. |
| Resend | Outbound email for vendor alerts (alert subjects use `firm_id` hash; no raw firm names in subject lines) | Subject + body of vendor alert emails | **None.** Alert content contains the `firm_id` hash + structured alert codes; no Client Data. |
| Slack (optional) | Vendor on-call channel for alert echo | Same content as Resend emails when this sink is enabled | **None.** Same carve-out as Resend. |
| Apple (future) | Code-signing certificate for client apps (not yet enrolled) | App bundle contents (which contain no Client Data) | **None.** |

6.3 The Processor will notify the Controller in writing not less than 30 days before engaging any new sub-processor or expanding the data exposure of an existing sub-processor. The Controller may object within 14 days; an unresolved objection allows the Controller to terminate the Agreement without penalty.

6.4 The Processor maintains the up-to-date sub-processor list at `docs/vendor-sop/vendor-sub-processors.md` in the codebase repository and provides a current copy on request.

---

## 7. Security

7.1 The Processor will implement and maintain appropriate technical and organisational measures to protect Client Data against unauthorised access, disclosure, loss, or destruction.

7.2 These measures include, as a minimum:
  (a) Encrypted storage of all data at rest (AES-256);
  (b) TLS 1.3 encryption for all network communication within the Controller's environment;
  (c) API key authentication with SHA-256 hashed tokens for all access to the Services;
  (d) Role-based access controls limiting system access to authorised users only;
  (e) Automated audit logging of all access events.

7.3 The Processor will notify the Controller of any security incident affecting the Services within 24 hours of becoming aware of it, and will provide a written incident report within 72 hours.

---

## 8. Data subject rights

8.1 The Processor will assist the Controller in responding to data subject requests under the UK GDPR (access, erasure, rectification, portability) within the Services architecture.

8.2 Given that Client Data is processed entirely on the Controller's premises and the Processor has no access to query content, most data subject requests will be fulfilled directly by the Controller. The Processor will provide technical assistance on request.

---

## 9. Liability

9.1 The Processor's aggregate liability to the Controller under or in connection with this Agreement (whether in contract, tort, or otherwise) shall not exceed the total fees paid by the Controller to the Processor in the 12 months preceding the event giving rise to the claim.

9.2 Clause 9.1 does not apply to: (a) death or personal injury caused by negligence; (b) fraud or fraudulent misrepresentation; (c) any liability that cannot be excluded by law.

9.3 Neither party shall be liable for indirect or consequential loss, loss of profit, loss of revenue, or loss of reputation, except to the extent caused by a breach of Clauses 3 or 4 of this Agreement.

### 9.4 Nature of AI-generated outputs and required human review

**9.4.1 The Services produce outputs (including but not limited to summaries, drafted text, retrieved passages, suggested authorities, and answers to questions) that are generated by automated language models. Such outputs are, in every case, AI-generated suggestions intended to assist a qualified human professional and are not, and must not be relied upon as, legal, financial, regulatory, or other professional advice.**

**9.4.2 The Controller is solely responsible for ensuring that every output produced by the Services is reviewed and verified by an appropriately qualified human professional before being acted upon, communicated to a client, filed with a court or regulator, or used as the basis for any decision affecting a third party.**

**9.4.3 The Processor accepts no liability whatsoever for any loss, damage, claim, sanction, or adverse outcome arising from the use of an AI-generated output that has not been independently reviewed and verified by a qualified human professional. This exclusion applies notwithstanding any other provision of this Agreement and is a fundamental term reflecting the nature of the technology supplied.**

9.4.4 The Processor will display, in the user interface of the Services, a visible, persistent disclaimer alongside every AI-generated response indicating that the response is AI-generated and must be verified before use. The Controller acknowledges that this disclaimer is provided for the benefit of end users and does not transfer the Controller's verification obligation to the Processor.

### 9.5 Operating environment maintained at supported version

**9.5.1 The Controller will not apply, install, or permit the application or installation of any operating system update, firmware update, or material configuration change to the Hardware without the prior written approval of the Processor. "Material configuration change" includes, without limitation, macOS major-version upgrades, replacement of the Python toolchain, or modification of the launchd service definitions installed by the Processor.**

**9.5.2 Breach of clause 9.5.1 voids the Service Level Agreement for the period during which the Hardware is operating outside the Processor's supported configuration. The Processor remains available to assist the Controller in returning the Hardware to a supported configuration on a best-effort basis but does not undertake to do so within any specific time window.**

9.5.3 The Processor maintains and publishes a list of supported operating system versions in its Standard Operating Procedure. The Processor will give the Controller not less than thirty (30) days' written notice before withdrawing support for a previously-supported version.

9.5.4 Security patches issued by Apple under macOS Rapid Security Response (or equivalent successor mechanism) are excluded from the requirement in clause 9.5.1 and may be applied at the Controller's discretion. The Processor will, on becoming aware of any Rapid Security Response that materially affects the Services, notify the Controller within five (5) business days.

### 9.6 Model substitution

9.6.1 The Processor may substitute the underlying open-source language model and/or embedding model used by the Services with an equivalent open-source model on not less than thirty (30) days' written notice to the Controller. "Equivalent" means a model of materially comparable parameter count, quantisation tier (Q8 or higher half-precision), and licence (open-weights with no telemetry). The notice will identify the new model, the rationale (security patch, capability improvement, withdrawal of upstream support, etc.), and the date of the proposed substitution. The Controller may object within fourteen (14) days; an unresolved objection allows the Controller to terminate the Services agreement without penalty under Clause 10.

## 10. Term and termination

10.1 This Agreement commences on the effective date and continues for the duration of the Services agreement.

10.2 On termination of the Services agreement, the Processor will (at the Controller's election) either: (a) transfer all Client Data on the Hardware to the Controller in a portable format; or (b) securely erase all Client Data from the Hardware and provide written certification of erasure within 10 business days.

10.3 The Processor will also remove the Hardware from the Controller's premises within 30 days of termination.
